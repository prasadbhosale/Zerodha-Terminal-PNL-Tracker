# Zerodha Terminal P&L Tracker

A real-time terminal dashboard for tracking your Zerodha positions and profit/loss, built with [Kite Connect API](https://kite.trade/docs/connect/v3/) and [Rich](https://github.com/Textualize/rich).

## Features

- Live-updating positions table, refreshed every few seconds
- Per-position: Symbol, Product (CNC/MIS/NRML), Quantity, Avg Price, LTP, Day P&L, Overall P&L
- Color coded — green for profit, red for loss
- Summary panel with total Day P&L and Overall P&L
- Demo mode to preview the UI without an API connection

## Requirements

- Python 3.10+
- A [Kite Connect](https://developers.kite.trade/) app (free to create)

## Setup

### 1. Clone and install dependencies

```bash
git clone https://github.com/your-username/Zerodha_Terminal_PNL_Tracker.git
cd Zerodha_Terminal_PNL_Tracker
pip install -r requirements.txt
```

### 2. Create a Kite Connect app

1. Log in at [developers.kite.trade](https://developers.kite.trade/)
2. Create a new app
3. Set the **Redirect URL** to `http://localhost:7676/api/callback`
4. Note your **API Key** and **API Secret**

### 3. Configure credentials

```bash
cp .env.example .env
```

Edit `.env` and fill in your values:

```env
KITE_API_KEY=your_api_key_here
KITE_API_SECRET=your_api_secret_here
KITE_ACCESS_TOKEN=your_access_token_here   # generated by get_token.py
REFRESH_INTERVAL=3
```

### 4. Generate an access token (run every trading day)

Kite access tokens expire at the end of each trading day. Run this each morning:

```bash
python get_token.py
```

This will:
- Open the Kite login page in your browser
- Start a local server to capture the redirect automatically
- Save the access token to your `.env`

### 5. Run the tracker

```bash
python tracker.py
```

Press `Ctrl+C` to quit.

## Demo Mode

To preview the UI with simulated data (no API needed):

```bash
python demo.py
```

## Files

| File | Description |
|---|---|
| `tracker.py` | Main live P&L tracker |
| `get_token.py` | Daily access token generator |
| `demo.py` | Demo mode with simulated positions |
| `.env.example` | Credential template |
| `requirements.txt` | Python dependencies |

## Configuration

| Variable | Default | Description |
|---|---|---|
| `KITE_API_KEY` | — | Your Kite Connect API key |
| `KITE_API_SECRET` | — | Your Kite Connect API secret |
| `KITE_ACCESS_TOKEN` | — | Generated daily by `get_token.py` |
| `REFRESH_INTERVAL` | `3` | Seconds between data refreshes |

## Security Note

Never commit your `.env` file. It is excluded via `.gitignore`. Only `.env.example` (with placeholder values) should be committed.
